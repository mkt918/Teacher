PHASE 1

Google Apps Script の準備

1

Google Apps Script を開く

ブラウザで以下のURLにアクセスします。Googleアカウントでログインしていない場合はログインしてください。

https://script.google.com
💡

普段使っているGoogleアカウント（Gmail）でOKです。学校のGoogleアカウントがあればそちらを推奨します。

2

新しいプロジェクトを作成する

「新しいプロジェクト」ボタンをクリックします。

script.google.com
Apps Script
＋ 新しいプロジェクト
最近のプロジェクト（なし）
3

プロジェクト名をわかりやすい名前に変更する

左上の「無題のプロジェクト」をクリックして名前を変更します。
例：「Teacher App 同期」

script.google.com
📄
Teacher App 同期
← ここをクリックして変更
4

コードを貼り付ける

エディタ内に最初から書いてあるコードを全部削除してから、gas/Code.gs の内容をコピーして貼り付けます。

📄 コード.gs
// ===== Teacher App クラウド同期 GAS =====
const SHEET_NAMES = {
  schedule: 'schedule',
  todos: 'todos',
  ...
};

function doPost(e) { ... }
function doGet(e) { ... }
⚠️

既存のコードは完全に削除してから貼り付けてください。
「function myFunction(){}」などが残っているとエラーになります。

📁

gas/Code.gs はTeacher Appのフォルダ内にあります。テキストエディタ（メモ帳など）で開いてコピーしてください。

5

上書き保存する

キーボードの Ctrl + S を押して保存します。

✅

ファイル名横の「●」マークが消えれば保存完了です。

PHASE 2

ウェブアプリとしてデプロイ（公開）する

6

「デプロイ」メニューを開く

画面右上の「デプロイ」ボタンをクリックし、「新しいデプロイ」を選択します。

script.google.com
▶ 実行
🚀 デプロイ ▾
📦 新しいデプロイ
🔄 デプロイを管理
7

種類を「ウェブアプリ」に設定する

「種類の選択」から「ウェブアプリ」を選びます。

新しいデプロイ
種類の選択
🌐 ウェブアプリ
8

アクセス設定を行う

以下の通りに設定します。

説明（任意） Teacher App 同期 v1
次のユーザーとして実行 自分（メールアドレス）
アクセスできるユーザー 全員
⚠️

「アクセスできるユーザー」は必ず「全員」にしてください。
「全員」にしないと、アプリからデータを送受信できません。

9

「デプロイ」ボタンをクリックする

設定が完了したら「デプロイ」ボタンをクリックします。

🔐

権限の承認を求められた場合：
「アクセスを承認」→ Googleアカウントを選択 →「詳細設定」→「Teacher App 同期（安全でないページ）に移動」→「許可」の順にクリックしてください。

10

ウェブアプリのURLをコピーする

デプロイ完了画面に表示される「ウェブアプリのURL」をコピーします。
このURLは後で使うので、メモ帳などに貼り付けておくと安心です。

🔗 https://script.google.com/macros/s/AKfy...（長い文字列）/exec
⚠️

このURLは他人に教えないでください。 URLを知っていれば誰でもデータにアクセスできます。

「設定を保存する」をクリックする

設定を保存する

14

「今すぐクラウドに保存」をクリックする

同じページ内に表示されるボタンをクリックして、A端末のデータをクラウドに初回アップロードします。

☁️ 今すぐクラウドに保存
☁️ クラウドから読み込む
⚠️

確認ダイアログが表示されます。
「この端末のデータをクラウドに上書き保存します」と表示されたら「OK」を押してください。

ヘッダーに 「☁️ 同期済 HH:MM:SS」 と表示されればアップロード完了！
PHASE 4

B端末（新しい端末）でデータを受け取る

📱 B端末（新規）
15

Teacher App を開く

B端末のブラウザでTeacher App を開きます。この時点ではまだデータは空の状態です。

💡

B端末は新しいブラウザや別のPCでも構いません。Teacher AppのURL（GitHub Pages）にアクセスするだけでOKです。

16

「基本設定」でGASエンドポイントURLを設定する

A端末と同じ手順（STEP 11〜13）で、同じGAS URL を貼り付けて「設定を保存する」します。

✅

GAS URLはA端末・B端末ともに同じURLを使います。

17

「クラウドから読み込む」をクリックする

設定保存後、「☁️ クラウドから読み込む」ボタンをクリックします。

☁️ 今すぐクラウドに保存
☁️ クラウドから読み込む
⚠️

確認ダイアログが表示されます。
「クラウドのデータでこの端末のデータを上書きします」と表示されたら「OK」を押してください。
B端末はまだ空なので問題ありません。

A端末のデータが読み込まれ、時間割・ToDo・行事が表示されれば完了！
完了後

設定完了後の自動同期について

✓

データ変更から2秒後に自動でクラウドに保存されます

時間割・ToDo・年間行事を変更するたびに、2秒後に自動でクラウドへ保存されます。手動操作は不要です。

変更中は 「☁️ 同期中...」 と表示されます
完了後は 「☁️ 同期済 HH:MM:SS」 に変わります
✓

別端末でページを開いたときに最新データを取得します

B端末でページを開くと、クラウドの方が新しければ自動でデータを取り込みます。

💡

ページを開きっぱなしのときは自動では更新されません。最新データを取得したい場合はページを再読み込み（F5）するか、「クラウドから読み込む」ボタンをご利用ください。

✓

同期されるデータの範囲

以下のデータがクラウドに同期されます。

✅

時間割（自分・クラス）

✅

ToDo リスト

✅

年間行事予定

✅

時間割メモ・変更

⚠️

出欠記録・座席表・名簿など他のデータはローカル保存のみです（クラウド非対応）。

困ったときは

トラブルシューティング

?

ヘッダーに「☁️ 同期エラー」と表示される

GAS URLが正しく設定されていないか、デプロイの設定が間違っている可能性があります。

🔧

① 基本設定のGAS URLが正しいか確認する
② GASのデプロイ設定で「アクセスできるユーザー：全員」になっているか確認する
③ GASを「新しいデプロイ」としてやり直し、新しいURLを設定し直す

?

デプロイ時に「承認が必要です」と何度も表示される

💡

「詳細設定」→「（プロジェクト名）（安全でないページ）に移動」→「許可」の順にクリックしてください。これはGoogleの安全確認であり、自分で作ったスクリプトなので問題ありません。

?

B端末でデータが空のままになる

🔧

① A端末で「今すぐクラウドに保存」を実行済みか確認する
② B端末の「クラウドから読み込む」ボタンを使って手動で取り込む
③ ヘッダーの同期ステータスが「同期済」になっているか確認する
